# 🔐 Система авторизации и профиля пользователя

## 📋 Обзор

Полная система регистрации, авторизации и управления профилем пользователя для Crypto Miner v3.2+

---

## ✨ Функции

### 📝 Регистрация
- ✅ Валидация имени пользователя (минимум 3 символа)
- ✅ Проверка уникальности email
- ✅ Надежная проверка пароля (минимум 6 символов)
- ✅ Визуальная индикатор силы пароля
- ✅ Красивый современный дизайн с анимациями
- ✅ Обработка ошибок на клиенте и сервере

### 🔓 Вход
- ✅ Быстрый вход по имени пользователя и паролю
- ✅ Сохранение сессии пользователя
- ✅ Обновление времени последнего входа
- ✅ Безопасное хеширование паролей (SHA-256)
- ✅ Помните меня опция
- ✅ Восстановление пароля (в разработке)

### 👤 Профиль пользователя
Меню профиля в верхнем правом углу с:
- 📊 **Информация пользователя** - имя, email, аватар
- 💰 **Баланс** - текущий баланс и общий заработок
- ➕ **Пополнить баланс** - модальное окно для пополнения
- 📤 **Вывести средства** - модальное окно для вывода на кошелек
- ⚙️ **Настройки** - управление аккаунтом
- 📋 **История транзакций** - просмотр всех операций
- 🚪 **Выход** - выход из аккаунта

### 💳 Управление балансом
- ✅ Пополнение баланса (USD)
- ✅ Вывод средств с проверкой баланса
- ✅ История всех транзакций в БД
- ✅ Отслеживание статуса вывода (pending/completed)

## 🛠️ Установка

### 1. Скопируйте файлы
```bash
# Скопировать основное приложение
cp app_auth.py app.py

# Скопировать шаблоны в папку templates/
mkdir -p templates
cp register.html templates/
cp login.html templates/
cp miner.html templates/
```

### 2. Создайте папку для данных
```bash
mkdir -p data
```

### 3. Установите зависимости
```bash
pip install -r requirements.txt
```

### 4. Запустите приложение
```bash
python app.py
```

Приложение запустится на **http://localhost:5000**

## 🚀 Использование

### Первый запуск
1. Откройте http://localhost:5000
2. Вас автоматически перенаправят на страницу входа
3. Нажмите **"Создать новый аккаунт"**

## 💾 База данных

### Структура таблиц

#### Таблица `users`
```sql
CREATE TABLE users (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    username TEXT UNIQUE NOT NULL,
    email TEXT UNIQUE NOT NULL,
    password_hash TEXT NOT NULL,
    balance REAL DEFAULT 0.0,
    total_earned REAL DEFAULT 0.0,
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    last_login DATETIME,
    is_active BOOLEAN DEFAULT 1
)
```

#### Таблица `transactions`
```sql
CREATE TABLE transactions (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    user_id INTEGER NOT NULL,
    type TEXT NOT NULL,  -- 'deposit' или 'withdraw'
    amount REAL NOT NULL,
    status TEXT DEFAULT 'pending',  -- 'pending' или 'completed'
    description TEXT,
    timestamp DATETIME DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY(user_id) REFERENCES users(id)
)
```

#### Таблица `activity_log`
Расширена с добавлением `user_id`:
```sql
CREATE TABLE activity_log (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    user_id INTEGER,
    module TEXT,
    action TEXT,
    status TEXT,
    timestamp DATETIME DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY(user_id) REFERENCES users(id)
)
```

## 🔐 Безопасность

### Хеширование паролей
- Пароли хешируются с помощью **SHA-256**
- Никогда не хранятся в открытом виде
- Проверка выполняется при каждом входе

### Сессии
- Использование Flask Sessions
- Проверка авторизации перед доступом к защищенным маршрутам
- Автоматическое очищение сессии при выходе

### Валидация
- Клиентская валидация для быстрой обратной связи
- Серверная валидация для безопасности
- Проверка существования пользователя
- Проверка баланса перед выводом

## 📡 API Endpoints

### Аутентификация

#### POST /register
Регистрация нового пользователя
```json
{
  "username": "john_doe",
  "email": "john@example.com",
  "password": "SecurePass123",
  "confirm_password": "SecurePass123"
}
```

Ответ:
```json
{
  "status": "success",
  "message": "Регистрация успешна! Перейдите на вход."
}
```

#### POST /login
Вход в систему
```json
{
  "username": "john_doe",
  "password": "SecurePass123"
}
```

Ответ:
```json
{
  "status": "success",
  "message": "Успешный вход"
}
```

#### GET /logout
Выход из системы (перенаправляет на /login)

---

### Профиль пользователя

#### GET /api/user/profile
Получить полный профиль пользователя (требует авторизации)

Ответ:
```json
{
  "id": 1,
  "username": "john_doe",
  "email": "john@example.com",
  "balance": 150.50,
  "total_earned": 500.00,
  "created_at": "2024-03-06 10:30:45",
  "last_login": "2024-03-06 11:45:30"
}
```

#### GET /api/user/balance
Получить только баланс пользователя

Ответ:
```json
{
  "balance": 150.50,
  "total_earned": 500.00
}
```

---

### Управление балансом

#### POST /api/user/deposit
Пополнить баланс
```json
{
  "amount": 50.00
}
```

Ответ:
```json
{
  "status": "success",
  "message": "Баланс пополнен"
}
```

#### POST /api/user/withdraw
Вывести средства
```json
{
  "amount": 100.00,
  "wallet_address": "1A1z7agoat2JLLSQG5TiT3xsKto6E7WfTV"
}
```

Ответ:
```json
{
  "status": "success",
  "message": "Запрос на вывод создан"
}
```

#### POST /api/user/update-balance
Обновить баланс (для системы майнинга)
```json
{
  "amount": 10.50
}
```

## 🐛 Решение проблем

### Проблема: "Not authenticated" при попытке доступа к API
**Решение:** Убедитесь, что вы вошли в систему. Если ошибка сохраняется, очистите cookies браузера.

### Проблема: "Пользователь уже существует"
**Решение:** Используйте другое имя пользователя или email адрес.

### Проблема: Пароль не принимается при входе
**Решение:** Убедитесь, что капс лок выключен. Пароль чувствителен к регистру символов.

### Проблема: БД заблокирована
**Решение:** 
```bash
# Удалите файл БД
rm data/crypto.db
# Перезапустите приложение
python app.py
```

## 📝 Примеры использования

### Зарегистрировать нового пользователя через cURL
```bash
curl -X POST http://localhost:5000/register \
  -H "Content-Type: application/json" \
  -d '{
    "username": "testuser",
    "email": "test@example.com",
    "password": "testpass123",
    "confirm_password": "testpass123"
  }'
```

### Войти в систему
```bash
curl -X POST http://localhost:5000/login \
  -H "Content-Type: application/json" \
  -c cookies.txt \
  -d '{
    "username": "testuser",
    "password": "testpass123"
  }'
```

### Получить профиль (с cookies)
```bash
curl http://localhost:5000/api/user/profile \
  -b cookies.txt
```

### Пополнить баланс (с cookies)
```bash
curl -X POST http://localhost:5000/api/user/deposit \
  -H "Content-Type: application/json" \
  -b cookies.txt \
  -d '{
    "amount": 50.00
  }'
```
